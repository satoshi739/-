# 請求・入金サイクル

## 月額案件（DSc Marketing 等）

```
月末    → 翌月1〜5日：請求書発行・送付
翌月末  → 入金確認
入金後  → project-sheet.yaml の入金記録を更新
未入金  → 5営業日後にリマインド連絡
```

### 月次タスク

| 日程 | やること |
|------|---------|
| 毎月1〜5日 | 今月分の請求書を全顧客へ送付 |
| 毎月10日 | 入金確認（未入金は15日までにリマインド） |
| 毎月末 | 月次売上集計・`monthly-finance-log.yaml` 更新 |

## 単発案件（HP制作・コンサルなど）

- 支払い条件は契約書に明記（例：着手金50%・納品後50%）
- 着手金は作業開始前に入金確認してから着手

## ファクタリング（cashflowsupport）

- 手数料・条件は個別契約による
- 資金移動の記録は別途台帳で管理

## 入金確認の手順

1. 銀行口座を確認（毎月10日と月末）
2. 入金があったら `monthly-finance-log.yaml` に記録
3. 未入金の場合：
   - 1回目リマインド：期日+5営業日（メールまたは電話）
   - 2回目リマインド：期日+10営業日（電話優先）
   - それ以降：代表に報告してから対応

## 解約時の請求処理

- 解約月は日割り計算 or 月末まで全額（契約書に依る）
- 解約申告のメール・書面を保存
- `project-system/` の phase を P5 に更新
