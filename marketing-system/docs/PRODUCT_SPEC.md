# プロダクト仕様（v1 — 販売可能な最小単位）

## 目的

「毎日、同じ品質の型でリール用の**設計から投稿文案まで**を出し切る」こと。  
v1 では **動画ファイルの自動レンダリングと全SNSへの一括投稿は必須にしない**（ツール・API・権限に依存するため）。代わりに、**編集者・投稿担当が迷わない成果物**を必ず出す。

## 想定ユーザー（あなたの会社）

- 自社マーケで毎日回す
- 同じ型を他社に渡して運用代行・ライセンス販売する

## v1 で「必ず」出る成果物（1トピック／1日＝1リール想定）

以下を **1セット** と定義する。ファイル名は運用で統一すること。

| # | 成果物 | 説明 |
|---|--------|------|
| A | リール設計書 | [`templates/reel-brief.md`](../templates/reel-brief.md) に沿った1本の指示書 |
| B | 秒構成つき台本 | フック〜CTA、想定秒数、テロップ行（読み上げ／画面テキストの区別） |
| C | 引用・要約ブロック | 出典、引用範囲の明示、要約1〜3行、注意書き（推奨） |
| D | キャプション複数案 | 短／中／長、プラットフォーム別に調整可能な形 |
| E | ハッシュタグ・検索意図 | 乱発しない。カテゴリ別に整理 |
| F | WordPress 向け下書き用テキスト | タイトル案、リード、本文骨子、内部リンク候補（任意） |
| G | 品質レビュー記録 | [`docs/QUALITY_BAR.md`](QUALITY_BAR.md) のチェック結果 |

## v1 の入力（クライアント／自社共通）

毎回、最低限これを埋める。

- **トピック**（1行）
- **ターゲット**（誰に向けるか）
- **トーン**（例：真面目／カジュアル／権威）
- **禁止事項**（表現、競合名、医療・金融など敏感領域）
- **出典の扱い**（引用可／要約のみ／オリジナル主張のみ）

## v1 の範囲外（営業で「まだ約束しない」）

- 特定ツールへの完全ワンクリック書き出し（未選定のため）
- 著作権フリー音源の自動選定のみに依存した商用利用保証（ライブラリ契約が別途必要）
- 各SNS API による無人投稿（審査・権限が案件ごとに異なる）
- 成果の数値保証（再生数など）

## v1 の差別化（売るときの芯）

- **再現性**：同じ入力形式・同じ設計書・同じ品質ゲート
- **説明責任**：引用・要約・注意書きをセットにする
- **拡張性**：後から編集ツール・投稿連携を「モジュール」として足せる

## バージョンの上げ方

- **v1.1**：選んだ1編集フロー（例：テロップのみテンプレ）に合わせたエクスポート形式を追加
- **v2**：選んだ1〜2チャネルへの「下書きまで」の自動投入を追加
