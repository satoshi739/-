# プロジェクトパイプライン

## フェーズ定義

```
[P1 キックオフ] → [P2 実行中] → [P3 確認・修正] → [P4 納品済/継続中] → [P5 完了/解約]
```

### P1 キックオフ
- トリガー：契約書署名完了
- やること：
  1. `project-sheet.yaml` を `projects/[案件番号]/` に起票
  2. キックオフミーティングを1週間以内に開催（`kickoff-agenda.md` 使用）
  3. 顧客の連絡先・素材提供方法・承認フローを確認
  4. Slackチャンネル or LINE グループ作成
- 完了条件：顧客と作業範囲・スケジュール・連絡方法が合意済み

### P2 実行中
- 月次：作業実施 → 週次で進捗共有（月額案件）
- 単発案件：マイルストーン設定済み
- ブロッカー発生時：24時間以内に顧客に報告・代替案提示

### P3 確認・修正
- 納品物を顧客へ提出
- 修正回数の上限は契約書に明記する（目安：2回まで）
- 修正内容はテキストで受け取る（口頭不可）

### P4 納品済・継続中
- 月額案件：毎月この状態を維持
- 単発案件：検収書（または承認メール）を取得して完了

### P5 完了・解約
- 完了の場合：最終報告書を作成・送付
- 解約の場合：解約理由をメモ → `customer-success-system/` に記録
- `finance-system/` の請求を停止処理

## 月次ルーティン（月額案件）

| タイミング | やること |
|-----------|---------|
| 月初1〜3営業日 | 先月の作業サマリー送付・当月計画確認 |
| 月中 | 作業実施・週次進捗共有 |
| 月末最終週 | 月次レポート作成（`monthly-report.md` 使用）|
| 月末 | 翌月請求書発行（`finance-system/` 連携）|
