# プロジェクト管理システム（Project System）

契約後の**キックオフ → 実行 → 納品 → 継続更新**までを管理する。

## 読む順番

1. [`docs/PIPELINE.md`](docs/PIPELINE.md) — フェーズ定義と進め方
2. [`docs/QUALITY_GATES.md`](docs/QUALITY_GATES.md) — 納品前の品質確認項目
3. [`templates/project-sheet.yaml`](templates/project-sheet.yaml) — 1案件のマスタ情報
4. [`templates/kickoff-agenda.md`](templates/kickoff-agenda.md) — キックオフミーティングの構成
5. [`templates/monthly-report.md`](templates/monthly-report.md) — 月次レポートのひな型

## ブランド別の主なプロジェクト種別

| ブランド | 典型的な案件 | 期間目安 |
|----------|--------------|----------|
| DSc Marketing | 月額SNS・LINE・Web運用 | 継続 |
| DSc Marketing | HP制作・リニューアル | 1〜3ヶ月 |
| cashflowsupport | ファクタリング相談・実行 | 1〜2週間 |
| UPJ | 事業設計コンサル | 3〜6ヶ月 |

## 案件番号ルール

```
[ブランド略称]-[YYYY]-[連番3桁]
例: DSC-2026-001 / CSF-2026-001 / UPJ-2026-001
```
