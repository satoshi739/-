# データモデル（店舗マスタ）

## 原則

- **1店舗 = 1 YAML ファイル**（例：`shops/shop-slug/profile.yaml`）
- 機密（パスワード・APIキー）は **YAML に書かない**。環境変数または秘密管理で別扱い
- 表示用と内部用を混ぜない（`public` ブロックに載るものだけを掲載用に出力）

## 推奨フィールド（最小）

| フィールド | 説明 |
|------------|------|
| `id` | 内部ID（英小文字・ハイフン） |
| `name_ja` / `name_kana` | 店名 |
| `category` | 業種タグ（検索・文案のトーン用） |
| `address` | 住所（郵便番号・都道府県・以降） |
| `phone` | 代表電話（E.164 推奨は任意） |
| `hours` | 曜日別営業時間（定休日含む） |
| `holiday_note` | 臨時休業・年末年始など自由記述 |
| `services` | 短い箇条書き（3〜7） |
| `usp` | 一言の強み |
| `cta` | 予約・来店の誘導（1つに絞る） |
| `links` | 公式サイト、LINE、地図、予約URL など key-value |
| `campaigns` | 期間付きキャンペーン（任意・配列） |
| `brand_voice` | トーン・禁止語（マーケと共通でもよい） |
| `channels` | この店が使っている掲載先のチェックリスト（論理名） |

## 拡張

- 複数店舗は `shops/` 下にディレクトリ分割
- 画像はパスまたはアセットIDのみ（バイナリは別ストレージ）
