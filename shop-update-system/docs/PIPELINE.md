# パイプライン（店舗更新）

## フェーズ 0 — 初回オンボーディング

1. 既存掲載の棚卸し（GBP、インスタ、HP、チラシ等）
2. `shop-profile.schema.yaml` を埋める（正の情報を確定）
3. `channels` に「どこに出しているか」を列挙

## フェーズ 1 — 日常更新（v1 の製品）

1. **変更依頼**（例：定休日変更、キャンペーン追加）
2. **マスタ YAML を編集**（差分が履歴で分かるよう Git 推奨）
3. **チェックリスト**（[`templates/update-run-checklist.md`](../templates/update-run-checklist.md)）で掲載先ごとに反映
4. **出力テキスト生成**（チャネル別にコピペ用ブロックを生成 — LLM またはスクリプト）
5. **監査ログ**（日時・担当・変更概要を1行記録）

## フェーズ 2 — 半自動

- YAML からテキストを機械的に組み立てるスクリプト（例：Node/Python）
- WordPress 下書き API など、**1チャネルずつ** 接続

## フェーズ 3 — 自動

- スケジューラ + 承認フロー（重要店舗は人の承認を挟む）

## 「ボタン一つ」への定義

| 段階 | ボタンの意味 |
|------|----------------|
| 今 | マスタ更新 + チェックリスト完了 + コピペ用ブロック出力 |
| 次 | スクリプト1発で「全チャネル用テキストをファイル出力」 |
| 将来 | 承認後に API で投稿・下書き |
