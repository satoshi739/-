# プロダクト仕様（店舗更新 — v1）

## 目的

お店の **営業時間・定休日・電話・住所・サービス説明・キャンペーン** などを、  
毎回バラバラに直さず、**マスタ1つ**から各掲載先へ反映できる状態にする。

## v1 で「必ず」約束できること

| # | 内容 |
|---|------|
| A | **店舗プロファイル**（YAML）という単一の正 |
| B | 更新時の **差分チェックリスト**（何をどこに反映するか） |
| C | **チャネル別の出力テキスト**（コピペ用：GBP用短文、インスタプロフィール用、WPお知らせ草案など） |
| D | 更新の **監査ログ**（いつ・誰が・何を変えたか）をテンプレで残す運用 |

## v1 で標準では約束しないこと

- 全チャネルを **API だけで無人更新**（アカウント権限・審査・レート制限が店舗ごとに異なる）
- 地図・検索の **順位保証**
- 他社が管理する媒体の **勝手な書き換え**

## v2 以降で足すモジュール（営業は「オプション」として）

- WordPress REST API で下書き投稿
- Google Business Profile API（OAuth・ロケーションID が必要）
- Meta Graph API（Instagram／Facebook）

## 差別化（営業の芯）

- **ブレないマスタ** → 担当交代・外注でも同じ型
- **反映先リストが可視化**される → 更新漏れが減る
- 自動化は **後から足せる**（マスタが先にある）
