# 営業用：店舗更新システムで売ること

## v1 で売っているもの

- **店舗情報のマスタ（YAML）** による単一の正
- 更新のたびに使う **反映チェックリスト**（掲載漏れ防止）
- **チャネル別のコピペ用文案**（GBP・SNS・ブログ向けに整形）
- 変更の **監査ログ** の型（いつ・誰が・何を）

## 言っていいこと

- 「掲載先が増えるほど更新漏れが出やすいので、**マスタから流す**形にします」
- 「最初は手動反映でも、**型ができてから** API 連携を足していきます」
- 「検索順位は保証しませんが、**情報の矛盾**は減らせます」

## 言わないこと

- 「全チャネルを無人で自動投稿し続ける」（権限とポリシーが店舗ごとに異なる）
- 「MEO対策で必ず上位」（保証はしない）

## デモで見せるもの

- 埋まった `profile.yaml`（架空店舗で可）
- 1回の更新で出力した **チャネル別テキスト**
- チェックリストに全部チェックが付いた例
