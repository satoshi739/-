# 契約更新チェックリスト

契約終了日の**1ヶ月前**に開始する。

## 顧客の状態確認（更新提案前）

- [ ] `client-health-sheet.yaml` の総合スコアを確認
- [ ] 過去3ヶ月の月次レポートを振り返り、成果のサマリーを準備
- [ ] 未払い・請求トラブルがないか確認（`finance-system/` 照合）
- [ ] アットリスクの兆候（`docs/AT_RISK.md`）がないか確認

## 更新提案の準備

- [ ] 現行プランの継続か、改訂プランを提案するかを決める
- [ ] 料金変更がある場合：変更理由と新料金を文書化
- [ ] 改訂版の提案書 or 更新確認メールを準備

## 顧客との更新面談（推奨）

- 実施タイミング：契約終了日の3〜4週前
- アジェンダ：
  1. 過去の成果振り返り（数値・実績）
  2. 顧客の現状と今後の課題ヒアリング
  3. 次期プランの提案・条件確認
  4. 更新意志の確認

## 更新決定後

- [ ] 更新契約書を電子契約で送付・署名取得
- [ ] `project-sheet.yaml` の `end_date` を更新
- [ ] `monthly-finance-log.yaml` に継続MRRとして記録
- [ ] 担当者へ引き続きよろしくお願いしますのメッセージ

## 更新辞退（解約）の場合

→ `offboarding-checklist.md` に移行
