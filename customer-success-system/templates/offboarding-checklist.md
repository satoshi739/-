# オフボーディングチェックリスト（解約・縮小時）

解約が確定したら、**顧客との関係を前向きに終わらせる**ことを最優先にする。
丁寧な対応が将来の再契約・紹介に繋がる。

## 解約確定後の連絡

- [ ] 解約受諾の確認メールを送付（電話で合意した場合も必ず文書化）
- [ ] 解約日・最終請求日を文書で確認
- [ ] 引き継ぎが必要な情報・アクセス権限を整理

## 業務の引き継ぎ

- [ ] 管理しているアカウント（Instagram / LINE / WordPress 等）の権限を顧客へ移管
- [ ] 作成した素材・ファイル（画像・文章・設計書）を顧客へ渡す
- [ ] 操作マニュアルが必要か確認（あれば簡易版を提供）
- [ ] ドメイン・サーバーの情報が顧客側で把握できているか確認

## 財務処理

- [ ] 最終請求書を発行・送付
- [ ] 入金確認後に `monthly-finance-log.yaml` の `churned_clients` に記録
- [ ] `project-sheet.yaml` の `phase` を P5・`billing.payment_status` を closed に更新

## 解約理由の記録（内部のみ）

- [ ] 解約理由を `project-sheet.yaml` の `notes` に記録
  - 予算 / タイミング / 効果不満 / 担当者交代 / 事業縮小 / 競合 / その他
- [ ] 再アプローチの可否・時期をメモ（6ヶ月後など）

## 最後のメッセージ

- [ ] 感謝のメッセージを送る（テンプレ不使用・一言でも自分の言葉で）
- [ ] 「また機会があればぜひ」を自然な形で伝える

---

> **Note:** 解約顧客を粗末に扱わない。紹介・再契約・SNSでの口コミは解約後も起きる。
