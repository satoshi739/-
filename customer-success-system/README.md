# カスタマーサクセス（Customer Success）

契約後の**定着・満足・更新・解約時の丁寧な引き継ぎ**までを管理する。  
`project-system/`（納品・フェーズ）と `finance-system/`（請求・入金）と並行して運用する。

## 読む順番

1. [`docs/LIFECYCLE.md`](docs/LIFECYCLE.md) — CS上のステージと project の対応
2. [`docs/AT_RISK.md`](docs/AT_RISK.md) — 離反の兆候と初動
3. [`templates/client-health-sheet.yaml`](templates/client-health-sheet.yaml) — 顧客健全性の1枚シート
4. [`templates/qbr-agenda.md`](templates/qbr-agenda.md) — 四半期レビュー（任意）
5. [`templates/renewal-checklist.md`](templates/renewal-checklist.md) — 更新・契約更新前
6. [`templates/offboarding-checklist.md`](templates/offboarding-checklist.md) — 解約・スコープ縮小時

## 鉄則

- 成果を**断定せず**、期待値は文書（提案書・契約）に照らして揃える
- 「解約しそう」を検知したら、**sales に無断で値引き連発**しない（方針は上長と一致）
- 顧客の声は `notes` に残し、マーケ・プロダクト改善に還流させる
